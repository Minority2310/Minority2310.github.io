# 简介

Nmap是一个开源的网络连接端扫描软件，用来扫描计算机开放的网络连接端。它可以确定哪些服务运行在哪些连接端，并且推断计算其运行哪个操作系统。它是网络管理员必用的软件之一，也用于评估网络系统安全
如今Nmap更是增加许多实用的插件，可以用来检测SQL注入，网页爬行，数据库密码检测，号称扫描器之王



# 标准格式

```bash
nmap <扫描类型> <选项> {目标范围}
```



# 目标说明

除了选项，所有出现在Nmap命令行上的都被视为对目标主机的说明。 最简单的情况是指定一个目标IP地址或主机名

虽然目标通常在命令行指定，下列选项也可用来控制目标的选择：

```bash
# 从列表中输入
-iL <filename>

# 随机选择目标
# 0意味着永无休止的扫描
-iR <hostnum>

# 排除主机/网络
--exclude <host1,host2,...>

# 排除文件中的列表
# 文件中排除目标用换行符、空格或制表符分隔
--excludefile <excludefile>
```



# 主机发现

任何网络探测任务的最初几个步骤之一就是把一组IP范围(有时该范围是巨大的)缩小为 一列活动的或者您感兴趣的主机。

由于主机发现需求是如此多样，因此Nmap提供了多种用于自定义所用技术的选项。主机发现有时称为`ping扫描`，但它远远超出了与无处不在的ping工具关联的简单`ICMP`回显请求数据包。

用户可以通过列表扫描`-sL`或禁用`ping`（完全跳过ping步骤`-Pn`），或使用任意组合的多端口`TCP SYN / ACK`，`UDP`，`SCTP INIT`和`ICMP探针`使网络参与。这些探针的目的是征求表明IP地址实际上处于**活动状态**（正在由主机或网络设备使用）的响应。在许多网络上，在任何给定时间只有一小部分IP地址处于活动状态

如果没有给出主机发现的选项，Nmap将发送`ICMP回显请求`(将`TCP SYN数据包`发送到`443端口`，将`TCP ACK数据包`发送到`80端口`)和`ICMP时间戳请求`。如果是`IPV6`，则省略`ICMP时间戳请求`，因为它不是ICMPv6的一部分。这些默认值等同于`-PE -PS443 -PA80 -PP`选项。一个例外是`ARP扫描`用于局域网上的任何目标机器。对于非特权Unix Shell用户，默认探测是使用`connect()`系统调用到达`80和443端口`的`SYN数据包`。

`-P*`选项(用于选择 ping的类型)可以被结合使用。即使指定了其它` -P*选项`，ARP发现(`-PR`)对于局域网上的目标而言是默认行为

**主机发现选项：**

```bash
# 列表扫描
# 确保目标IP正确
-sL

# 无端口扫描(ping扫描)
# 探测存活主机
-sn

# 无ping
# 跳过主机发现阶段，对目标进行高强度扫描
-Pn

# TCP SYN Ping，默认80端口
# 进行SYN探测，如果目标主机存活，则会发一个RST包扼杀正在建立的连接
-PS<port list(22-25,80,443)>

# TCP ACK Ping，默认80端口
# 发送的是ACK请求包，不会建立起真正的连接，防火墙封锁SYN报文时采用此方法
-PA<port list(22-25,80,443)>

# UDP Ping，默认40125端口
# 多数情况下该端口都是关闭的，存活的主机可能会发送一个ICMP错误
-PU<port list(22-25,80,443)>

# SCTP INIT Ping，默认80端口
# 目标端口关闭，ABORT块响应；端口开放，INIT-ACK响应
-PY<port list(22-25,80,443)>

# ICMP Ping类型
-PE
-PP
-PM

# IP协议Ping，默认ICMP(protocol 1)、IGMP(protocol 2)和IP-in-IP(protocol 4)协议
# TCP(protocol 6),UDP(protocol 17),SCTP(protocol 132)
-PO<protocol list>

# 无ARP或ND Ping
--disable-arp-ping

# 主机的跟踪路径
# 适用于除连接扫描(-sT)和空闲扫描(-sI)外的所有扫描类型
--traceroute

# 不用域名解析
# 不要对找到的活动IP地址进行DNS解析
-n

# 为所有目标解析域名
# 始终对目标IP地址执行反向DNS解析
-R

# 扫描每个解析的地址
--resolve-all

# 使用系统DNS解析器
--system-dns

# 用于反向DNS查询的服务器
--dns-servers <server1,server2,...>
```



# 端口扫描

## 端口扫描基础

虽然Nmap这些年来功能越来越多，但端口扫描程序仍然是它的核心功能。简单的命令`nmap <target>`扫描主机上的1000个TCP端口。

Nmap把端口分为六个状态，这些状态并非端口本身的性质，而是描述Nmap怎样看待它们。例如， 对于同样的目标机器的`135/tcp端口`，从同网络扫描显示它是开放的，而跨网络作完全相同的扫描则可能显示它是` filtered(被过滤的)`。

### Nmap所识别的6种端口状态

- **open(开放的)**

应用程序正在该端口接收`TCP 连接`，`UDP数据报`或`SCTP关联`，发现这一点常常是端口扫描 的主要目标。非安全扫描可能对开放的端口也感兴趣，因为它们显示了网络上那些服务可供使用。

- **closed(关闭的)**

关闭的端口对于Nmap也是可访问的(它接受Nmap的探测报文并作出响应)， 但没有应用程序在其上监听。它们可以显示该IP地址上(主机发现，或ping扫描)的主机正在运行，也对部分操作系统探测有所帮助。 

- **filtered(被过滤的)**

Nmap无法确定端口是否打开，因为数据包筛选会阻止其探针到达端口。过滤可以来自`专用防火墙设备`，`路由器规则`或基于主机的`防火墙软件`。有时候它们响应`ICMP错误消息`如类型3代码13(无法到达目标: 通信被管理员禁止)，但更普遍的是过滤器只是丢弃`探测帧`， 不做任何响应。 这迫使Nmap重试若干次以访万一探测包是由于网络阻塞丢弃的。 这使得扫描速度明显变慢。

- **unfiltered(未被过滤的)**

未被过滤状态意味着端口可访问，但Nmap不能确定它是开放还是关闭。 只有用于映射防火墙规则集的`ACK扫描`才会把端口分类到这种状态。 用其它类型的扫描如`窗口扫描`，`SYN扫描`，或者`FIN扫描`来扫描未被过滤的端口可以帮助确定端口是否开放。

- **open|filtered(开放或未被过滤的)**

当无法确定端口是开放还是被过滤的，Nmap就把该端口划分成 这种状态，开放的端口不响应就是一个例子。没有响应也可能意味着报文过滤器丢弃了探测报文或者它引发的任何响应。因此Nmap无法确定该端口是开放的还是被过滤的。 `UDP`，`IP协议`， `FIN`，`Null`，和`Xmas`扫描可能把端口归入此类。

- **closed|filtered(关闭或被过滤的)**

当Nmap无法确定端口是关闭还是过滤时，将使用此状态。它只会出现在`IP ID空闲扫描`中。

## 端口扫描技术

虽然Nmap努力产生正确的结果，但请记住所有结果都是基于目标机器(或者它们前面的防火墙)返回的报文的。这些主机也许是不值得信任的，它们可能响应以迷惑或误导Nmap的报文。 更普遍的是**非RFC兼容**的主机以不正确的方式响应Nmap探测。`FIN`，`Null`和`Xmas扫描` 特别容易遇到这个问题。

除了`UDP扫描`(-sU)和`STCP扫描`(-sY,-sZ)可以与任何一种`TCP扫描`结合使用外，其他扫描方法一次只能使用一种。

`-s*`为基本选项格式(*代表大写字母)

```bash
# TCP SYN扫描(半开放扫描)
# 很少有系统会把这种扫描记入系统日志，需要root权限
-sS

# TCP connect()扫描(全开放扫描)
# 会在目标主机的日志记录大批的链接请求以及错误信息
-sT

# TCP NULL，FIN和Xmas扫描(隐秘扫描)
# 被扫描系统如果符合TCP RFC文本，当端口关闭时，任何不包含SYN，RST，或者ACK位的报文会返回一个RST；而当端口开放时，应该没有任何响应
# Null扫描，不设置任何标志位(TCP标准头是0)
-sN
# FIN扫描，只设置TCP FIN标志位
-sF
# 圣诞扫描，设置FIN，PSH，和URG标志位
-sX

# UDP扫描，不可靠
# DNS，SNMP，和DHCP服务(注册端口为53，161/162，67/68)运行在UDP协议上
-sU

# SCTP INIT扫描
# SCTP INIT扫描与TCP SYN扫描的SCTP等效
-sY

# TCP ACK扫描
# 用于映射防火墙规则集，确定它们是否是有状态的以及对哪些端口进行了过滤
-sA

# TCP窗口扫描
# 不支持它的系统通常会返回所有端口closed
-sW

# TCP Maimon扫描
# 该技术与NULL，FIN和Xmas扫描完全相同，只是探针为FIN/ACK
-sM

# 自定义TCP扫描
--scanflags

# SCTP COOKIE ECHO扫描
-sZ

# 空闲扫描
# 这种高级的扫描方法允许对目标进行真正的TCP端口盲扫描
-sI <zombie host[:probeport]>

# IP协议扫描
# 可以确定目标机支持哪些IP协议(TCP，ICMP，IGMP...)
-sO

# FTP弹跳扫描
-b <FTP relay host>
```

## 扫描规格和扫描顺序

Nmap还提供用于指定要扫描哪些端口以及扫描顺序是随机的还是顺序的选项。默认情况下，Nmap为每个协议扫描最常见的1,000个端口

```bash
# 仅扫描指定的端口
# 扫描协议类型TCP、UDP、SCTP或IP协议
-p <U:53,111,T:21-25,80>

# 从扫描中排除指定的端口
--exclude-ports <port ranges>

# 快速扫描(有限端口扫描)
-F

# 不要按随机顺序扫描端口
-r

# nmap-services以大于给定比率的比率扫描文件中的所有端口
# <ratio>必须在0.0到1.0之间
--port-ratio <ratio>

# 在排除由指定的所有端口之后，扫描文件中<n>找到的最高比率的端口
# <n>必须为1或更大
--top-ports <n>
```
